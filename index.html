Modèle IA

Quelle intelligence pour votre chatbot ?

Le modèle LLM est le moteur du chatbot. Vous payez uniquement ce que vous consommez.

💡 Comment ça fonctionne : Chaque conversation consomme des tokens (unités de texte). Vous achetez une clé API auprès d'un fournisseur et êtes facturé selon votre usage réel. Estimez ~100 000 tokens/jour pour un usage scolaire actif.

Ce que dit le RGPD

Points de vigilance pour une école traitant des données d'étudiants.

📋
Registre de traitement

Inscrire le chatbot dans le registre RGPD de l'école avec la finalité et les données traitées.

Obligatoire
📝
Information des utilisateurs

Afficher une notice d'information claire avant utilisation du chatbot.

Obligatoire
🤝
DPA fournisseurs

Signer un accord de traitement des données avec chaque fournisseur de service API externe.

Si API externe
🗑️
Durée de conservation

Définir et appliquer une durée de rétention des conversations (ex : 6 mois).

Recommandé
🔐
Sécurisation des accès

Accès admin Dify protégé par double authentification (2FA) et mots de passe forts.

Transferts hors UE

Privilégier les hébergeurs et LLM européens pour éviter les transferts de données hors UE.
